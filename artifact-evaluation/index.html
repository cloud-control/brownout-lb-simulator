Artifact - Modeling of Request Cloning in Cloud Server Systems
 using Processor Sharing

Software requirements: Docker, Matlab for data pre-processing
Hardware requirements: Standard Linux PC for evaluation, access to some sort of computing infrastructure is
highly reccomended for running the actual experiments done in the paper.

The artifact contains the cloning simulator, raw data and data preprocessing, and plotting of the data in LaTex as
displayed in the paper.

Running the artifact docker container

Everything needed to run the components of the artifact is included in the form of a dockerfile. To build the
associated image, simply run

> docker build -t artifact .

By starting the container, and sharing the folders for the plots and cloning simulator, results can easily be
accessed from the host. To start the container and run bash interactively with the shared folders, do

> docker run -it -v $(pwd)/plots:/work/plots -v $(pwd)/cloning-simulator:/work/cloning-simulator artifact /bin/bash

Evaluating the simulator

The simulator is contained in the /work/cloning-simulator directory and is directly executable from the artifact
container.

The main script of the simulator, "simulator.py" runs a single simulation for the given input parameters and
supplied scenario file. To get a detailed help on the available arguments for the simulator, run

> python simulator.py  --help

The scenario files in "scenarios/" adds servers of desired types, and sets simulation
parameters.

Larger simulations of e.g. parameter sweeps is most easily done through scripting. An example on how to run
the simulator in a scripted fashion is available in "sim_example.py" and its corresponding scenario file
"scenarios/clone-example.py". To run the example script, simply type

> python sim_example.py

The results are stored in the directory "results_example/". Since the cloning-simulator is a shared directory the
output results can be accessed on the host computer directly.

Recreating the plots

The plots seen in the paper can be generated in LaTeX from the processed raw data. From the "/work/plots" 
directory on the artifact container, simply run

> pdflatex main.tex

Since the plots directory is shared with the host, the resulting pdf-file is easily accessed.
